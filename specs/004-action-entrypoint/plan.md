# Plan — Action entrypoint

## Approach

Single-phase implementation on this worktree. The surface is wider than PR 003 but has well-defined seams:

- **Pure orchestration core** (`src/action.ts`) — reads inputs, calls already-tested `fetchContributions` + `renderCometSVG`, writes files. Zero novel logic; this is glue.
- **Imperative side-effect shell** — `@actions/exec` git sequence. Ugly but small (≤ 20 lines), well-captured by unit tests that stub exec.
- **Build tooling** — ncc is a drop-in bundler; `build:action` is a one-line script.
- **CI gate** — `check:dist` is a 10-line shell check.

The renderer and data layer stay frozen. PR 004 composes them; it does not alter them. The only fresh surface is the Action wiring itself plus the committed bundle discipline.

## Why not `/ralph` or direct executor

- `/ralph` loop: human-in-the-loop signals are cheap on this PR — we want a `code-reviewer` pass before push (per PR 002/003 pattern), not an autonomous loop.
- Direct executor (what we'll use): decisions are fixed below, spec is concrete, ncc+orphan-push is mechanical, so a scoped executor + one reviewer pass + Codex review is the right fit. Matches PR 002 (3 Codex cycles) and PR 003 (awaiting review) shape.

## Open-question resolutions

The parent request to `/plan` enumerated eight open questions. Resolutions, recorded here so the executor does not reopen them:

1. **Orphan push: inside action vs in workflow YAML?** → **Inside action**. User installs a single Action; no companion workflow lines to maintain. Matches Platane/snk.
2. **Reduced variant generation: renderer flag vs post-process vs separate fn?** → **Existing `animated: false`**. `src/renderer.ts:482` already strips `<animate*>` elements when `animated=false`. No renderer change, no new abstraction.
3. **Year boundaries: does action.ts need to pass window args?** → **No**. `src/data.ts:146` computes the rolling 371-day UTC window internally. `action.ts` calls `fetchContributions(username, token)` with no third arg.
4. **ncc output path: `dist/` vs alternative?** → **`dist-action/`**. Deviates from target doc verbatim (`dist/`) to avoid conflict with Vercel's `dist/` (already gitignored, regenerated by `pnpm build`). Update the target doc in the same PR. Parallel to existing `dist-renderer/`.
5. **Error handling: setFailed vs throw?** → **`core.setFailed(err.message)`**, no re-throw, no `process.exit`.
6. **Testing: unit vs `act` integration?** → **Unit at orchestration boundary**. `tests/action.test.mjs` stubs `globalThis.fetch` (same pattern as PR 003's `tests/data.test.mjs`) and stubs `@actions/exec.exec` via module monkey-patching. No `act`/Docker dep in MVP.
7. **Local smoke runner?** → **Yes, `scripts/run-action.mjs`**. Mirrors `scripts/fetch-contributions.mjs`: reads `GITHUB_TOKEN`, username from argv, runs the full flow including renderer, writes SVGs to `sample-out/`, prints planned git commands without executing push. Safe on author's machine.
8. **Dist PR size?** → **Acceptable** (100–500 KB per Platane/snk baseline). `.gitattributes linguist-generated` + `-diff` makes GitHub collapse the bundle in PR view. No size gate in MVP; add one only if a future bundle regresses past ~1 MB.

### Step 2 — `src/action.ts`

```ts
import { readFile, writeFile, mkdir } from "node:fs/promises";
import { tmpdir } from "node:os";
import { join } from "node:path";
import * as core from "@actions/core";
import { exec } from "@actions/exec";
import { fetchContributions } from "./data.js";
import { renderCometSVG } from "./renderer.js";
import { DARK_THEME } from "./themes.js";

export async function run(): Promise<void> {
  try {
    const username = core.getInput("username", { required: true });
    const token = core.getInput("token") || process.env.GITHUB_TOKEN || "";
    const reduced = (core.getInput("reduced") || "true") !== "false";
    const branch = core.getInput("branch") || "comet-graph";

    if (!token) {
      throw new Error("token input or GITHUB_TOKEN env var is required");
    }

    const days = await fetchContributions(username, token);

    const animated = renderCometSVG(days, {
      theme: DARK_THEME,
      animated: true,
    });

    const workdir = await mkdtemp(join(tmpdir(), "comet-graph-"));
    await writeFile(join(workdir, "comet.svg"), animated, "utf8");
    const files = ["comet.svg"];

    if (reduced) {
      const reducedSvg = renderCometSVG(days, {
        theme: DARK_THEME,
        animated: false,
      });
      await writeFile(join(workdir, "comet-reduced.svg"), reducedSvg, "utf8");
      files.push("comet-reduced.svg");
    }

    await pushOrphan({ workdir, branch, token, files });

    core.info(`Pushed ${files.join(", ")} to ${branch}`);
  } catch (err) {
    const msg = err instanceof Error ? err.message : String(err);
    core.setFailed(msg);
  }
}

async function pushOrphan(args: {
  readonly workdir: string;
  readonly branch: string;
  readonly token: string;
  readonly files: readonly string[];
}): Promise<void> {
  const opts = { cwd: args.workdir };
  await exec("git", ["init", "--quiet"], opts);
  await exec("git", ["config", "user.name", "comet-graph-bot"], opts);
  await exec(
    "git",
    ["config", "user.email", "comet-graph-bot@users.noreply.github.com"],
    opts,
  );
  await exec("git", ["checkout", "--orphan", args.branch], opts);
  await exec("git", ["add", ...args.files], opts);
  await exec(
    "git",
    ["commit", "--quiet", "-m", `chore: regenerate ${args.files.join(", ")}`],
    opts,
  );
  const repo = process.env.GITHUB_REPOSITORY;
  if (!repo) throw new Error("GITHUB_REPOSITORY env var missing");
  const url = `https://x-access-token:${args.token}@github.com/${repo}.git`;
  await exec(
    "git",
    ["push", "--force", "--quiet", url, `HEAD:${args.branch}`],
    opts,
  );
}

if (import.meta.url === `file://${process.argv[1]}`) {
  void run();
}
```

Notes:

- Export `run()` so tests call it directly; self-invoke at module bottom so ncc-bundled entry is executable.
- `mkdtemp` comes from `node:fs/promises`. Plan note: actual file uses `mkdtemp` + `mkdir` correctly.
- Token redaction: once push URL is built, the variable is confined to the exec call. `@actions/exec` does not echo argv at info level. For paranoid users, set `core.setSecret(args.token)` before any exec call — add it.
- `process.env.GITHUB_REPOSITORY` is set by the GitHub Actions runner (format `owner/repo`). Local smoke runner sets it explicitly.

### Step 6 — `scripts/run-action.mjs` (local smoke)

Key difference from the real Action: the `push` step is replaced with `console.log` of the command it _would_ run. Sample shape:

```js
#!/usr/bin/env node
import { spawnSync } from "node:child_process";
process.env.INPUT_USERNAME = process.argv[2] ?? "";
process.env.INPUT_TOKEN = process.env.GITHUB_TOKEN ?? "";
process.env.INPUT_REDUCED = process.env.INPUT_REDUCED ?? "true";
process.env.INPUT_BRANCH = process.env.INPUT_BRANCH ?? "comet-graph";
process.env.GITHUB_REPOSITORY = process.env.GITHUB_REPOSITORY ?? "local/smoke";
process.env.COMET_DRY_RUN = "1"; // reserved flag read by action for local smoke

if (!process.env.INPUT_USERNAME) {
  console.error("Usage: node scripts/run-action.mjs <username>");
  process.exit(1);
}

const build = spawnSync("pnpm", ["run", "build:renderer"], {
  stdio: "inherit",
});
if (build.status !== 0) process.exit(build.status ?? 1);

// Import tsc-compiled JS, not ncc bundle — avoids rebuilding dist-action/ on every smoke.
const mod = await import("../dist-renderer/action.js");
await mod.run();
```

The action's `pushOrphan` checks `process.env.COMET_DRY_RUN === "1"` and logs instead of executing. That's a one-line branch inside `pushOrphan`. Recorded here so the executor doesn't miss it.

To make `dist-renderer/action.js` available, extend `tsconfig.json` to still include `src/action.ts` (it already globs `src/**/*.ts`, so this is automatic). The `build:renderer` tsc pass picks up action.ts and emits `dist-renderer/action.js`. This is the local-dev-only path; the real Action runs the ncc bundle at `dist-action/index.js`.

Testing mechanics:

- **`globalThis.fetch` stub** — same pattern as `tests/data.test.mjs`. Returns the `tests/fixtures/graphql-sample.json` payload.
- **`@actions/exec.exec` stub** — module monkey-patch using Node's built-in test module mocks (`--experimental-test-module-mocks` if needed; otherwise export `exec` through a file-local `execFn` variable that tests can reassign).
- **`@actions/core` stub for inputs** — `core.getInput` reads `INPUT_<NAME>` from `process.env`; tests set those env vars. `core.setFailed` is captured by wrapping it via test setup (either a local `setFailedCalls: string[]` array populated by stubbing `core.setFailed`, or observing the process exit code).
- **tempdir** — real `fs.mkdtemp` into OS tmp, cleaned in afterEach.

Target: ≥ 8 cases. Keep each test <20 lines. `afterEach` restores `globalThis.fetch`, exec stub, env state.

## Risks

- **Exec argv injection via `username` / `branch` inputs.** If a malicious workflow caller passes `username: "--upload-pack=evil"` or similar, it could confuse git. Mitigation: validate inputs against a strict allowlist regex in `action.ts` (username: `/^[A-Za-z0-9][A-Za-z0-9-]{0,38}$/`, branch: `/^[A-Za-z0-9._/-]{1,100}$/`) before passing to exec. Reject with `setFailed` on mismatch. Add to test matrix.
- **Token leak via git error messages.** `git push` on auth failure may echo the URL containing `x-access-token:<token>`. Mitigation: `core.setSecret(args.token)` before any exec; `@actions/core` scrubs secrets from subsequent logs. Add a test asserting `core.setSecret` is called with the token.
- **Bundle drift between source and committed dist-action.** Author forgets to rerun `build:action`. Mitigation: `check:dist` CI gate runs ncc fresh and compares to committed; fails PR on mismatch. Pre-push hook (optional, future) could also run it.
- **ncc TS mode picks wrong tsconfig.** ncc reads `tsconfig.json` by default. Verify the emitted bundle: (a) does not contain `export` at top level (it's ES5-style CJS unless `--target esnext`), (b) respects `strict` from our tsconfig (should — ncc uses our config). Mitigation: ncc `--target es2022` flag + smoke-run of the bundle.
- **Node 20 runtime divergence on GitHub runners.** Runners may upgrade to Node 22+ before we're ready. Mitigation: `runs.using: node20` is a Github-supported LTS-ish tag; GitHub will transitively bump, but ncc output is ES2022 compatible up through Node 24.
- **Bundle size creep past ~1 MB.** Future dep additions could bloat. Mitigation: if `dist-action/index.js` exceeds 1 MB, add a gate. Not in this PR — YAGNI.
- **Lockfile churn for 3 new deps.** `pnpm install` may add transitive deps. Mitigation: run `pnpm install` once, review `pnpm-lock.yaml` diff for surprises (e.g., native deps that require build steps on install).
- **`check-feature-memory` gate blocks push if `specs/004-action-entrypoint/` incomplete.** Fine — that's the gate working. Ensure all three files land in the first commit.
- **Codex inline-comment pinning** (`feedback_codex_inline_pinned_to_head`): verify verdict from Codex summary comment, not inline-comment count or `commit_id`.
- **pnpm lockfile + frozen install in CI** (`feedback_pnpm_new_dep_flow`): run bare `pnpm install` locally first, commit updated lockfile, then CI's `--frozen-lockfile` passes.
